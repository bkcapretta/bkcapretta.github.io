School 2017 Projects

This past Spring semester, I took two computer science classes where I learned way more than I expected to learn. In my first class listed below, all projects were completed with a partner. In the second class, all projects were completed independetly except for the final group project.

Projects in Machine Structure and Assembly Language Programming

  • Files, Pictures, and Interfaces: reported the average brightness of an image file, and implemented readaline that reports similarities in file data
  • Interfaces, Implementations, and Images: implemented a two-dimensional, polymorphic, unboxed array using Hanson's C interfaces and implementations, and to then used that implementation to identify Sudoku solutions
  • Cache and Locality: implemented blocked two-dimensional arrays, and then used it to evaluate the performance of image rotation using three different array-access patterns with different locality properties
  • Integer and Logical Operations: given a PPM image, compressed the image by transforming color spaces and discarding info using lossy compression. There was a lot of packing and unpacking of binary data, work with two's complement, and floating-point arithmetic.

    Here's a funny picture of what the compressed image looked like half-way through the project:


  • Analysis of AMD64 Assembly Code: read, analyzed, and understood a half-dozen or so C procedures given access only to an executable binary. In the process of doing so, learned how to use two debuggers (GDB and DDD)
  • A Universal Virtual Machine: wrote a software implementation of a simple virtual machine
  • Code Improvement Through Profiling: used profiling tools (like kcachegrind) to improve the performance of the universal machine emulator
  • Assembly-Language Programming: implemented a calculator that used Reverse Polish Notation in assembly language

Projects in Web Programming

  • The Black Car Service (Client Side): used Google Maps' API to retrieve neighbors' coordinates and display their locations on a map
  • The Black Car Service (Server Side): created get and post requests to enter and keep track of passengers and vehicles
  • Security Vulnerability: identified three security issues with another student's Black Car Service project, including but not limited to cross-site scripting, lack of rate limiting, and out-of-bounds input.

Summer 2016 Projects

Last summer I worked at the Center for Engineering Education and Outreach (CEEO) where I learned about:

  • Arduinos
  • Raspberry Pis
  • GrovePi (Dexter Industries)
  • Wifi and Bluetooth
  • Internet of Things
  • If-This-Then-That model
  • Javacript and Python

For my first project of the summer, I was tasked with making an inventory system for the lab. I used Google Spreadsheet's powerful tool, the Script Editor, to create this project. I learned Javacript while creating it and learned that Google has some neat built-in functions.

Here is a look at the interface. See more about it on Github.

Later on in the summer, I worked with the GrovePi (while my partner worked with the EV3) and the idea of the Internet of Things. The purpose was to take physical inputs to produce virtual outputs with the devices, and vice versa. One example I created posted tweets when a button was pressed on the GrovePi. Another small project I worked livestreamed all the current tweets and a light sensor would flash when a specific hashtag appeared.
